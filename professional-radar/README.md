# Professional Radar

Scripts for the Product Radar publishing pipeline on the dkharlanau.github.io site.

## Scripts

### professional_radar_site_update.py

Takes accepted radar candidates and creates or updates site collection files (`_radar/` and optionally `_news/`).

**Workflow:**
1. Dry-run to preview changes
2. Inspect preview and confirm candidates
3. Apply with `--limit 1` (or more) to create/update files
4. Run validation and build

**Usage:**

```bash
# Preview what would be created
python3 professional-radar/scripts/professional_radar_site_update.py --dry-run

# Actually create/update files (limit to 1 for safety)
python3 professional-radar/scripts/professional_radar_site_update.py --apply --limit 1

# Use a custom candidate directory
python3 professional-radar/scripts/professional_radar_site_update.py --source-dir datasets/ai-business-signals --dry-run
```

**Safety:**
- `--dry-run` is the default
- `--apply` must be explicit
- `--limit N` caps the number of candidates processed
- Low-confidence candidates are rejected automatically
- `--skip-validation` is available but not recommended

### professional_radar_validate.py

Validates radar and news content for quality, correctness, and safety.

**Checks:**
- Frontmatter completeness (required fields)
- Confidence level (reject low, warn on medium)
- Content quality (key points, practical impact, source attribution)
- File naming conventions (`YYYY-MM-DD-slug.md`)
- No sample/draft files in production collections
- Atlas topic routing (optional)

**Usage:**

```bash
# Content validation only
python3 professional-radar/scripts/professional_radar_validate.py --all --stage site

# Content + Jekyll build
python3 professional-radar/scripts/professional_radar_validate.py --all --stage build
```

## Candidate Format

Candidates are JSON files with the following structure:

```json
{
  "title": "Signal Title",
  "date": "2026-05-15",
  "source": "Source Name",
  "source_url": "https://example.com",
  "confidence": "high",
  "topics": ["topic_one", "topic_two"],
  "short_summary": "Brief description.",
  "key_points": ["Point 1", "Point 2"],
  "practical_impact": "What this means for practitioners."
}
```

## Generated Content

Files are written to `_radar/` and `_news/` with:
- Valid YAML frontmatter
- Required metadata (layout, title, date, source, source_url, confidence, topics)
- Markdown body with key points and practical impact sections
- Source attribution footer
- "Generated by Professional Radar" disclaimer

## Collections

The site uses two Jekyll collections for radar content:
- `_radar/` — signal radar entries
- `_news/` — dated news items (with permalink override)

Both collections are configured in `_config.yml` with `output: true`.
