<section class="section note-detail"><p class="eyebrow">ABAP</p><h1>MCP for ABAP development</h1><p class="lead">ABAP tooling is useful when it keeps the normal ADT, authorization, activation, and transport controls intact.</p><div class="note-body"><p>The official ADT MCP Server is local HTTP and uses a bearer token generated by ADT. Its transport is local; the underlying ABAP actions can still be privileged. Enable only the tools required for analysis, keep development and production separate, and require a reviewer for generated changes.</p><p>For a source inspection task, collect object name, package, transport context, syntax-check result, and ATC result. Do not let an agent activate or release a transport without a named human decision.</p></div></section>
