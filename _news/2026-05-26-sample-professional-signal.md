<h2>SAP clean-core guidance reinforces side-by-side extensibility</h2>

<p>SAP has continued to emphasize side-by-side extensibility as the preferred path for S/4HANA customizations, linking it to lower long-term TCO and easier upgrade readiness. This is not a new policy, but the messaging has become more explicit in 2025–2026 guidance.</p>

<p>For AMS teams, the practical effect is clearer: custom code inside the core increases support surface area and upgrade risk. Side-by-side extensions (BTP, CAP, external services) keep the core stable and make support handovers cleaner.</p>

<p>Watch for: whether vendor implementation partners default to in-app extensions anyway, and whether AMS contracts explicitly account for clean-core compliance.</p>
