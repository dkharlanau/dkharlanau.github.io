Understanding Master Forms and Content Forms

Master Forms

  • Definition: Templates that contain common design elements like logos, headers, and footers.

  • Purpose: Ensure a consistent look and feel across all forms.

  • Variants: Can include different layout variants (e.g., for different countries or brands).

Content Forms

  • Definition: Templates that contain the dynamic business data specific to a document.

  • Relationship with Master Forms: Reference master forms to include standard design elements.

  • Customization: Focus on the specific content and data fields needed.

Standalone Forms

  • Definition: Forms that combine both master and content into a single template.

  • Limitations: Do not support fragment reuse, making maintenance less efficient.

Scripting and Form Logic

Scripting Languages

  • JavaScript: For complex logic and interactions.

  • FormCalc: For simple calculations and expressions.

Events in Forms

Scripts can be attached to various events to control when they execute:

  • initialize: Runs when the form is initialized, after data merging.

  • calculate: Executes after data merge, before display; ideal for calculations.

  • form:ready: Occurs after merging and before pagination; can dynamically hide or show objects.

  • layout:ready: Happens after layout and pagination; useful for accessing master page objects.

Working with XDP Files

What are XDP Files?

  • XML Data Provider (XDP): A file format used by Adobe LiveCycle Designer for form templates.

  • Contents: Includes layout design and can reference data structures.

Using XDP Files

  1. Download the Template: From the Maintain Form Templates app, download the form template as a ZIP file.

  2. Extract the Files: The ZIP file contains an XDP file and an XSD file (which defines the data structure).

  3. Edit in Adobe LiveCycle Designer: Open the XDP file to modify the form layout.

  4. Bind Data Fields: Use the XSD file to bind data fields to the form.
