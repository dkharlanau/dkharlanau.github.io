SAP flexible workflow

The flexible workflow is a general function that allows defining condition-based workflows for business processes. The list of flexible workflows increases with the new releases and the information about the use case, configuration and authorization are and will be provided in the component's help documentation.

To have a quick look at the functionality we will take the popular process to release purchasing documents in Procurement: set up automatic, one-step, or multi-step approval workflows for purchase orders. It is possible to assign one or more approvers to each step and define approvers by role, users, or teams as approvers. The approvers can then approve or reject their work items in the app My Inbox. Information for the scenario can be found by the link Flexible Workflow for Purchase Orders.

Configuration

There are few steps to activate workflow to be ready for use:

  1. Scenarios are not visible in the Fiori Workflow Editor app until they have been activated in the back-end system. SAP NetWeaver> Application Server> Business Management> SAP Business Workflow> Flexibler Workflow> Scenario Activation. Activate WS00800238  WFL_for_PO Workflow for Purchase Order.
  2. Set document types that are used for Flexible workflow. Materials Management> Purchasing> Flexible Workflow for Purchase Orders> Activate Flexible Workflow for Purchase Orders.
  3. Maintain workflow tasks that need to be included in the task filter together with their names and decision keys SAP NetWeaver> SAP Gateway Service Enablement> Content> Workflow Settings> Maintain Task Names and Decision Options.

HR-BP-User sincronization

It is the prerequisite to use HR and BP to assign the task of releasing PO for a user.

Create a Personnel with the transaction PA30, assign a user to the Personnel number with the communication type, create a business partner with the transaction BP and assign an Identification number using ID type HCM001 with the Personnel number created before. At the end run report /SHCM/RH_SYNC_BUPA_EMPL_ SINGLE to synchronize created master data.

Integration into Purchasing document

Releasing purchasing orders happens via processing items in My Inbox. For each new PO where the criteria are met, a workflow will be started. The status of releasing is visible for PO with the Fiori App Manage purchase orders

Summary

With SAP flexible workflow it is easier to create and manage workflow scenarios. The extended documentation is provided in the component's documentation at sap.help.com. More and more scenarios will be added in the future as pre-built workflows for standard business processes where it makes sense to have them. The major part of configuration can be done by a key user itself. Complexity is reduced by simplifying step's type: it can be user decision or activity ( background and dialog, with or without smart forms ). It is more easily to setup email notifications with Email Templates.
