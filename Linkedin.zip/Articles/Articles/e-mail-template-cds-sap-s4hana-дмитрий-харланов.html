E-Mail-Template with CDS in SAP S/4HANA

Created on 2021-12-21 18:00

Published on 2021-12-21 18:34

It is the common desire to be aware of the current state, status of the critical processes where reaction for an issue resolution should be quick or where to be convenient a notification is necessary.

As an SAP consultant, I anticipate probable solutions to inform a user via sending an email

  1. With SAP Business workflow is set to create a task and send an email notification via event creation.
  2. A custom program that runs periodically or is triggered by an event ( by processing an output type ) to build and send an email dynamically using CL_BCS. W
  3. To create a log via the program's run and make it available per the user's request.
  4. ?

With SAP S4 on-premise or cloud Email template CDS is available to build an email quicker and easier. It relates to option #2, but on this side, it simplifies the process of email creation.

* CDS (Core data services) is a layer above "pure database" in order to define semantically-enriched data models.

How can I access Email templates and from which parts does it consist?

In ABAP Workbench (SE80): Create> More> E-Mail template. Using one of the pre-delivered Email templates ( SD_SLS_DEFAULT_EMAIL_TEMPLATE Sales Document Default Email Template ), we can see a possible use for it. 

In the picture below, you can see the structure of an Email template. It contains the header part where it is necessary to specify a CDS view and the Body part: the Subject and Email body with linking variables from CDS view, Translations. It is also possible with the special workaround to include pictures into the email body.

(!) The main restriction and disadvantage: it is only possible to pass a single line as the entry for processing. It means, if we need to list sales documents items as well with the sales document header, the approach will not work. And it is clear, to pre-build such function to work with several associations of CDS views, or with several sources requires a compex framework to cover it into the HTML structure.

What is about the coding?

Using the Email template SD_SLS_DEFAULT_EMAIL_TEMPLATE with CDS view, it is necessary to use cl_smtg_email_api for passing the name of the Email template and the key to retrieve the entry from the CDS View ( ~ a sales document header details ).

Here is the short list of lines needed to prepare and send an email.

DATA: lo_api      TYPE REF TO if_smtg_email_api

  TRY.
      DATA(lo_message) = NEW cl_bcs_message( ).

" Get email template
      lo_api = cl_smtg_email_api=>get_instance( 
               iv_template_id = 'SD_SLS_DEFAULT_EMAIL_TEMPLATE' ).
      
" Which sales order we will emailing?
      DATA(lt_data_key) = VALUE if_smtg_email_template=>ty_gt_data_key( 
                          ( name = 'SALESDOCUMENT' 
                            value = '0000000102' ) ).

      lo_api->render_bcs_message( io_bcs_message  = lo_message
                                  iv_language     = 'E'
                                  it_data_key     = lt_data_key  ).

" Add recipient
      lo_message->add_recipient( 'myemail@gmail.com' ).

" Set send immediately flag
      lo_message->set_send_immediately( iv_immediately = abap_true ).
      DATA(lt_error_recipients) = lo_message->send( ).


" Email errors
    CATCH cx_bcs_send INTO DATA(lo_bcs_error). 
"Email template errors
    CATCH cx_smtg_email_common INTO DATA(lo_stmg_error). 
  ENDTRY.

What is as a result?

In the end, an email is built and prefilled with variables, it contains a list of recipients, a subject, and the content. Everything that is needed to notify a responsible person about the status/event.
