Embedded analytics in SAP S4/HANA

Analytics is the term used for outside-in views, independent of the data sources needed. The focus of embedded analytics is on how to collect and present data available usually on the IT. The advantage of embedded analytics is that data for analytical processing as related transactional data are already available in the same system, and we can avoid the negative aspects of data duplication of data necessary for load into an external system. In addition, using the same data for transactions and analysis permits real-time access, avoiding the need for additional space or data modeling. SAP S/4HANA embedded analytics is a framework that enables users to rapidly model, create, and share their views, reports, KPIs, and apps.

Journey of Data from Tables to Fiori apps

To present the data for business users, the data should be created in relational database, pre-process and enriched with semantic data, exposed with throug the communication channel to UI. Therefore, we have three main layers:

Data layer: Virtual Data Models in Core Data Services

The core data services concept lets define database views independent of the application platform. CDS views are managed via ABAP and they comprises advanced means of writing data, handling transactions, and access control.

  • Associations, on a conceptual level, replacing joins with simple path expressions in queries
  • Annotations to enrich the data models with additional (domain-specific) metadata
  • Expressions, used for calculations and queries in the data model
  • Authorizations for CDS views

Virtual Data Models is the next term and they are based on CDS views and follow consistent implementation patterns and naming rules, which are based on business semantics. The key principle of the VDM is to build a semantic layer on top of the existing SAP S/4HANA database model, hiding its technical details.

Communication layer: Data Access Protocol

SAP Gateway, an integral part of SAP NetWeaver, enables connecting devices, environments, and platforms to SAP systems efficiently and securely. It uses the globally renowned OData protocol.

Presentation: Fiori

SAP Fiori launchpad is a role-based, personalized, and real-time aggregation point for business applications and analytics. It runs on multiple devices, using responsive web design, and is deployable on multiple platforms and serves as the platorm for using analitycs apps.

KPI Reporting (  ~ Drill down report )
Creating KPI 
A KPI group allows you to combine KPIs that are similar in terms of their definition and the data they use. All KPIs have to be assigned to a KPI group The most important thing for the definition of the KPI is the selection of a data source. 
Creating Drilldown Report
As soon as the KPI is saved and activated, the creation of a drilldown report can start from within the KPI.

Overview Pages ( ~ Dashboard )
Overview Page works as a dashboard. Overview Page provides a functionality to visualize data in an attractive and efficient way by adding different types of cards. 
Application header: Provides a description of the area for which this application provides an overview (for example, sales or procurement).
Smart filter: Provides application-level filters for changing the levels of data displayed in the cards.
Cards: A card is a smart component that uses UI annotation to render its content. Each card is bound to a single entity set in a data source. A card may display a donut or bar chart, or a table. 
Example: My Sales Overview
