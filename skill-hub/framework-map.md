  <section>
    <h2>How to use this map</h2>
    <ol>
      <li>When someone mentions a framework concept ("We need data governance"), find the concept in the map.</li>
      <li>Open the corresponding Skill Hub skill.</li>
      <li>Follow the working method to produce an artifact, not a definition.</li>
      <li>Use the decision rules to avoid generic recommendations.</li>
      <li>Record the outcome in the appropriate template.</li>
    </ol>
    <p>This map is not exhaustive. It covers the most common framework-to-skill mappings. If a framework area is missing, treat it as a gap and use the <a href="/skill-hub/business-analysis/gap-analysis-working-skill/">Gap Analysis Working Skill</a> to decide whether to add it.</p>
  </section>

    <h2>Verification status and limitations</h2>
    <p>This is a public working interpretation. It is not endorsed by DAMA International, IIBA, The Open Group, or SAP. Framework editions change; skills may need updating when frameworks are revised. The map focuses on practical skills, not comprehensive framework coverage. Some framework areas have no corresponding skill yet.</p>
