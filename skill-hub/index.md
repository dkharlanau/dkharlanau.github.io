<section class="section">
  <header class="section-heading">
    <h2>What Skill Hub is</h2>
  </header>
  <p>Skill Hub transforms professional knowledge and enterprise frameworks into <strong>usable work skills</strong>. Every page is designed to help someone do real work: diagnose a problem, ask better questions, structure unclear information, produce a useful artifact, review quality, make a decision, or guide an AI agent toward operational output.</p>
  <p>This is not a place to read about what data governance is. This is a place to learn <strong>how to diagnose missing data ownership and turn it into a governance action plan</strong>.</p>
  <p>This is not a place to read about event-driven architecture as a concept. This is a place to learn <strong>how to decide whether an event should exist, who owns it, what its contract is, and how failures are monitored</strong>.</p>
</section>

<section class="section">
  <header class="section-heading">
    <h2>Who this is for</h2>
  </header>
  <ul>
    <li><strong>Enterprise consultants</strong> who need structured methods for ambiguous client situations.</li>
    <li><strong>Business analysts</strong> who turn vague stakeholder complaints into requirements, assumptions, risks, and acceptance criteria.</li>
    <li><strong>Data consultants</strong> who diagnose data quality failures and build governance that sticks.</li>
    <li><strong>Solution architects</strong> who map capabilities, review designs, and record decisions with context.</li>
    <li><strong>Integration architects</strong> who decide between APIs and events, define contracts, and handle failures.</li>
    <li><strong>SAP AMS consultants</strong> who triage incidents, perform root cause analysis, and capture operational knowledge.</li>
    <li><strong>AI agents</strong> that need structured instructions to produce useful enterprise artifacts instead of generic explanations.</li>
  </ul>
</section>

<section class="section">
  <header class="section-heading">
    <h2>How to use Skill Hub</h2>
  </header>
  <ol>
    <li><strong>Pick a skill group</strong> that matches your current work situation.</li>
    <li><strong>Open a skill page</strong> and read the "When to use this skill" section.</li>
    <li><strong>Follow the working method</strong> step by step. Use the questions, decision rules, and templates provided.</li>
    <li><strong>Produce the deliverables</strong> listed on the page. Do not stop at understanding.</li>
    <li><strong>Use the quality checklist</strong> to review your output before sharing it.</li>
    <li><strong>Combine skills</strong> when a situation spans multiple domains. For example, a data quality incident may require Data Quality Root Cause + Stakeholder Analysis + Change Impact Analysis.</li>
  </ol>
</section>

<section class="section">
  <header class="section-heading">
    <h2>How AI agents should use Skill Hub</h2>
  </header>
  <p>AI agents should treat Skill Hub as an <strong>operational instruction set</strong>, not a knowledge base for summarization. See the <a href="/skill-hub/agent-usage-guide/">Agent Usage Guide</a> for detailed instructions.</p>
  <p>In short: choose a skill, ask for missing context, separate facts from assumptions, produce artifacts, and avoid generic framework language.</p>
</section>
