  <h1>Use AI tools safely and effectively for development work.</h1>
  <p class="lead">Skills for packaging repository context, designing tasks for coding agents, reviewing AI-generated code, selecting the right tool, and using MCP integrations without losing engineering control.</p>
</section>

<section class="section">
  <header class="section-heading">
    <h2>What this group covers</h2>
  </header>
  <p>AI-assisted development skills help you delegate coding and analysis work without handing off accountability. They provide structured methods for preparing repo context, writing narrow agent tasks, reviewing agent output, choosing tools by workflow fit, and running MCP servers with clear boundaries.</p>
  <p>These skills are cross-domain. They apply when you use Repomix, Claude Code, Codex CLI, Cursor, Copilot, Aider, CodeRabbit, MCP servers, or similar tools. Each page includes a working method, decision rules, templates, checklists, and agent instructions.</p>
</section>

<section class="section">
  <header class="section-heading">
    <h2>When to use this group</h2>
  </header>
  <ul>
    <li>You want to package a repository so an AI model can audit it or plan work.</li>
    <li>You are delegating a coding task to an agent and need it to stay narrow and reviewable.</li>
    <li>You need to review AI-authored changes before merge.</li>
    <li>You are choosing between Repomix, terminal agents, IDE agents, MCP, or code review tools.</li>
    <li>You are connecting MCP servers to an agent and need to define permission boundaries.</li>
  </ul>
</section>
